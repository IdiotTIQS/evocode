# Product Positioning

## What EvoCode Is Not

Getting positioning right requires being explicit about what EvoCode is not. These distinctions matter because the market is crowded with tools that superficially resemble EvoCode but represent fundamentally different bets.

### Not an AI Code Generator

AI code generators (GitHub Copilot, Cursor, Codeium) are autocomplete systems. They predict the next token or the next block of code based on what is already on the screen. They operate at the file level, have no model of the overall system, and produce output that the developer must review, integrate, and maintain.

EvoCode does not generate code fragments. It generates complete, verified, contextually-grounded changes to a real codebase with a full understanding of what is already there.

### Not a Low-Code Platform

Low-code platforms (Retool, OutSystems, Mendix) lower the floor of software development by constraining what you can build. They trade expressiveness for speed. They are walled gardens — fast inside the box, helpless outside it.

EvoCode has no walls. It targets real production codebases, real frameworks, and real deployment environments. There are no constraints on what can be built.

### Not a Code Completion Tool

Code completion tools assist a developer while they are typing. They are reactive, synchronous, and subordinate to the developer's workflow. The developer is still the author.

EvoCode is not reactive and not subordinate. It acts on an intent, builds a plan, and executes that plan. The developer is the decision-maker, not the typist.

## What EvoCode Is

### An Autonomous Software Engineering Platform

EvoCode is a platform where software engineering is performed by agents, not by people. Developers submit intent. Agents plan, implement, test, and review. The platform manages the full lifecycle of a change from requirement to verified code.

This is not a tool. It is a platform — a system with persistent state, multi-agent coordination, a knowledge graph of the codebase, and a continuous evolution loop.

### An Agent-Native Application Platform

EvoCode is designed from the ground up for agents. The architecture, the data models, the APIs, and the workflows all assume that the primary actor is an autonomous agent, not a human operating a GUI. Humans interact with the platform through intent and review, not through code editors.

Each agent in the platform has a clearly defined role, bounded responsibility, and operates through explicit interfaces. Agents collaborate through the Control Plane, which orchestrates their execution and manages shared state.

### An Operating System for Autonomous Software Engineering

Just as an operating system abstracts hardware and provides a runtime for applications, EvoCode abstracts software engineering mechanics and provides a runtime for agents. It handles:

- Project understanding and knowledge graph maintenance
- Task scheduling and agent dispatch
- Memory, context, and retrieval for agents
- Verification and safety gates
- Audit trail and change history

Teams plug their projects into EvoCode and get autonomous engineering capacity in return.

## The Market Category

EvoCode creates a new market category: **Autonomous Software Engineering**. It is adjacent to AI coding tools but distinct from them in the same way that a database is adjacent to a spreadsheet — they both handle data, but one is a tool and one is infrastructure.

The defining characteristic of this category is the **intent-to-code loop**: a fully automated pipeline from natural language requirement to verified, deployed code change, with human checkpoints at consequential decisions.

---

## Competitive Moat

EvoCode's durable advantages are:

1. **Project-level understanding** — agents operate on a knowledge graph of the codebase, not on isolated files
2. **Multi-agent coordination** — specialized agents collaborate under a shared plan, producing higher quality output than a single generalist model
3. **Continuous evolution** — the platform maintains state across changes, enabling incremental improvement rather than one-shot generation
4. **Verification loop** — changes are tested and reviewed before they are applied, making autonomous engineering safe enough for production
